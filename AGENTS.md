# AGENTS.md

适用范围：本文件夹内的“源素库 / 虚拟素材网站 / AI 数字素材交易平台”项目。

## 1. 项目定位

这是一个面向中国大陆 AI 视频、游戏、短剧广告和虚拟内容制作方的数字素材交易平台。第一版目标是 Web 网站 + 管理后台，支持人物、物件/道具、场景图片素材的认证上传、审核上架、付费购买、下载授权、收益记录和后台管理。

## 2. Codex 每次开始任务前必须读取

按顺序读取：

1. `PROJECT_START.md`
2. `docs/README.md`
3. `docs/项目总控.md`
4. 与任务相关的 `docs/` 文件
5. `docs/决策记录.md`
6. 如果任务来自 GitHub Issue，读取对应 Issue 内容

如果文档之间出现冲突，以 `docs/决策记录.md` 的最新记录和 Robert 的最新明确指令为准；仍不清楚时先问 Robert。

## 3. 工作原则

- 用中文和 Robert 沟通，除非 Robert 用英文提问。
- 保持真实、客观、准确；无法确认的内容标记为“待确认”，不要编造。
- 一次只推进一个明确任务，不把无关优化混进来。
- 不擅自删除文件、重置代码、清空数据或泄露密钥。
- 不把 GitHub、Notion、腾讯云部署、腾讯云 COS、Vercel、微信支付、支付宝、认证接口等外部系统当作已接通；必须先确认当前状态。
- 若需求不合理、不准确或有逻辑问题，先指出并请 Robert 确认。

## 4. 任务执行流程

1. 复述本次任务目标和不做范围。
2. 读取相关资料。
3. 如资料不足，列出需要 Robert 确认的问题。
4. 修改文档或代码。
5. 运行必要检查；如果没有代码或无法运行，说明原因。
6. 汇报修改内容、验收方式、风险和建议下一步。
7. 如果任务状态、计划或重要决策发生变化，同步更新 `docs/项目总控.md`、`docs/任务清单.md`、相关 docs，以及已接入的 Notion 总控页或 GitHub Issue。
