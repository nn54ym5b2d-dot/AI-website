# 测试验收清单

版本：v1.3
日期：2026-07-13

## 1. 使用方式

每完成一个开发任务，至少检查对应功能是否满足本文件的验收点。若暂时没有代码，本文件作为后续开发的验收依据。

## 2. 产品资料验收

- `docs/PRD.md` 能准确描述产品定位、目标用户和 MVP 范围。
- `docs/业务规则.md` 覆盖上传、认证、审核、购买、授权、收益和状态规则。
- `docs/页面清单.md` 能指导前台、个人中心和后台页面搭建。
- `docs/角色权限.md` 能明确不同角色能做什么、不能做什么。
- `docs/决策记录.md` 区分已确认和待确认问题。

## 3. 上传者流程验收

- 上传者不能无邀请码激活上传权限。
- 上传者可以创建人物、物件/道具、场景素材。
- 上传者可以上传原文件和预览图。
- 人物素材需要上传必要证明材料；物件/道具和场景素材不强制要求必要证明材料。
- 提交素材时进入认证上传费支付流程。
- 初审未通过时可以记录退款。
- 初审通过后可以进入认证状态。
- 认证完成后，管理员可以录入证书编号、凭证并保存认证证书文件。
- 所有素材，包括人物/物件/场景，必须获得认证证书后才可上架。

## 4. 购买下载流程验收

- 用户可以浏览、搜索、筛选素材。
- 用户可以查看素材详情、价格、认证状态和预览信息。
- 首版人物和场景素材统一显示 50 元，物件/道具素材统一显示 10 元。
- 未登录用户不能直接购买。
- 一个订单可以包含多个素材。
- 支付成功后，每个素材生成独立授权记录。
- 支付成功后，用户可以获取限时下载链接。
- 原始文件地址不能直接公开。
- 下载行为有记录。

## 5. 授权流程验收

- 授权记录包含购买用户、素材编号、订单编号、授权说明、授权时间和认证状态。
- 用户可以在个人中心查看自己的授权记录。
- 后台可以查看授权记录。
- 授权永久有效，不自然到期；授权状态支持有效、已撤销。

## 6. 收益流程验收

- 支付成功后，上传者收益记录增加。
- 首版每笔素材购买按上传者 80%、平台 20%、外部观察员 0% 记录收益。
- 上传者只能查看自己的收益。
- 管理员可以查看全部收益记录。
- 财务管理员可以查看收益和支付记录。
- 财务管理员可以发起和处理退款，且操作进入日志。
- 第一版不要求自动提现。

## 7. 后台权限验收

- 超级管理员拥有全部权限。
- 运营管理员不能修改角色和系统级设置。
- 财务管理员不能审核素材、修改素材内容、录入认证信息或修改价格。
- 外部观察员只能只读查看平台上传量、下载量、收益汇总和合作方分成相关数据。
- 外部观察员首版分成比例和金额显示为 0。
- 外部观察员不能导出任何后台数据。
- 外部观察员不能查看用户手机号、邮箱、证件信息、支付账户、完整支付流水号、素材原文件地址或下载原始链接。
- 关键后台操作进入操作日志。

## 8. 安全与合规验收

- 原始素材文件不公开暴露。
- 用户下载入口默认有效 365 天；每次实际下载使用短时 COS 签名 URL。
- 后台接口必须检查角色权限。
- 敏感信息不向外部观察员展示。
- 微信支付和支付宝回调需要验签。
- 不在代码中写入密钥、token、密码。

## 9. Robert 手动验收建议

每个阶段完成后，Robert 可以按以下方式验收：

1. 先看页面是否符合业务理解。
2. 再按上传者、购买用户、管理员、外部观察员分别走一遍流程。
3. 检查不该看到的信息是否被隐藏。
4. 检查不该操作的按钮是否不可用或不可见。
5. 检查订单、授权、收益记录是否正确产生。
