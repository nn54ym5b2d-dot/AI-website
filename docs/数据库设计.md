# 数据库设计细化版

版本：v2.0
日期：2026-07-13
状态：T007 细化版，等待 Robert 审核

## 1. 设计结论

第一版数据库使用 PostgreSQL。T007 只细化数据模型，不连接真实数据库、不创建生产数据表、不写完整业务接口。

推荐后续实现方式：**Prisma + PostgreSQL**。

原因：

- Prisma schema 对新项目更直观，适合 Robert 后续阅读和 Codex 维护。
- Prisma Migrate 能管理迁移记录，方便回滚和审查。
- 当前业务以结构化表关系为主，Prisma 足够覆盖用户、素材、订单、授权、收益和权限。
- 对复杂统计、部分索引、物化视图或高阶 SQL，可在后续用原生 SQL migration 补充。

暂不建议第一版直接使用 Drizzle 作为主方案。Drizzle 更贴近 SQL，但对当前新手管理和快速推进不如 Prisma 直观。

## 2. 全局建模原则

- 主键统一使用 `uuid`。
- 金额统一用整数分保存，例如 `amount_cents`，币种字段默认 `CNY`。
- 所有核心表保留 `created_at`、`updated_at`；需要软删除的表增加 `deleted_at`。
- 文件表只保存腾讯云 COS 的 `bucket`、`region`、`object_key`、哈希和元数据，不保存公开原始文件地址。
- 原始素材、人物证明材料、认证证书文件默认私有；预览图可按后续 COS/CDN 策略公开或半公开。
- 外部观察员只能看汇总表或脱敏字段，不能直接查用户、支付流水、原文件、下载链接、内部日志。
- 后续接口层必须按角色检查权限，数据库字段设计不能替代权限控制。

## 4. 用户与身份

### users

平台用户基础表。一个用户可以同时是购买用户、上传者、管理员或外部观察员，具体角色放在角色表中。

| 字段 | 类型建议 | 说明 |
|---|---|---|
| `id` | uuid pk | 用户 ID |
| `display_name` | text | 显示名称 |
| `email` | text nullable unique | 邮箱，敏感字段 |
| `phone` | text nullable unique | 手机号，敏感字段 |
| `avatar_url` | text nullable | 头像地址，可选 |
| `primary_login_method` | auth_provider | 主要登录方式 |
| `status` | user_status | 用户状态 |
| `last_login_at` | timestamptz nullable | 最近登录时间 |
| `created_at` | timestamptz | 创建时间 |
| `updated_at` | timestamptz | 更新时间 |
| `deleted_at` | timestamptz nullable | 软删除时间 |

索引：

- `unique(email)`，忽略 null。
- `unique(phone)`，忽略 null。
- `index(status, created_at)`。

外部观察员不可见：`email`、`phone`、登录时间、删除状态。

## 5. 素材、文件、审核与认证

### assets

素材主表。

| 字段 | 类型建议 | 说明 |
|---|---|---|
| `id` | uuid pk | 素材 ID |
| `uploader_profile_id` | uuid fk uploader_profiles.id | 上传者 |
| `asset_type` | asset_type | person、object、scene |
| `title` | text | 素材标题 |
| `description` | text nullable | 素材说明，可选 |
| `review_status` | asset_review_status | 审核状态 |
| `listing_status` | asset_listing_status | 上架状态 |
| `certification_status` | certification_status | 认证状态 |
| `price_cents` | integer | 当前售价，单位分 |
| `currency` | text | 默认 CNY |
| `rejection_reason` | text nullable | 驳回原因 |
| `submitted_at` | timestamptz nullable | 提交审核时间 |
| `reviewed_by_user_id` | uuid fk users.id nullable | 审核管理员 |
| `reviewed_at` | timestamptz nullable | 审核时间 |
| `listed_at` | timestamptz nullable | 上架时间 |
| `created_at` | timestamptz | 创建时间 |
| `updated_at` | timestamptz | 更新时间 |
| `deleted_at` | timestamptz nullable | 软删除时间 |

业务约束：

- `listing_status = listed` 时，必须满足 `review_status = approved` 且 `certification_status = certified`。
- `asset_type = person` 时，必须存在至少一个 `asset_files.file_type = person_proof`。
- 所有素材最终上架前必须存在认证证书文件。

索引：

- `index(asset_type, listing_status, certification_status)`。
- `index(uploader_profile_id, created_at)`。
- `index(review_status, created_at)`。

| 字段 | 类型建议 | 说明 |

| `access_scope` | file_access_scope | 访问范围 |
| `cos_bucket` | text | COS Bucket |
| `cos_region` | text | COS 地域 |
| `cos_object_key` | text | COS object key，不是公开 URL |
| `file_hash` | text | 文件哈希 |

## 10. 敏感字段和外部观察员可见范围

| 数据类别 | 外部观察员是否可见 | 说明 |
|---|---|---|
| 平台上传量、上架量、认证量 | 可见 | 只看汇总 |
| 下载次数、下载趋势 | 可见 | 只看汇总 |
| 订单数量、订单金额、退款金额 | 可见 | 只看汇总 |
| 净收益、平台分成、上传者分成 | 可见 | 只看汇总 |
| 合作方分成基数、比例、预计分成、已结算、待结算 | 可见 | 只看与自己相关记录 |
| 用户手机号、邮箱、证件信息 | 不可见 | 敏感个人信息 |
| 支付账户、完整支付流水号、回调明细 | 不可见 | 支付敏感信息 |
| COS object key、原文件路径、下载 token、临时 URL | 不可见 | 文件安全信息 |
| 人物证明材料、认证证书文件 | 不可见 | 权利/认证敏感文件 |
| 内部操作日志 | 不可见 | 后台安全与审计信息 |

```text
users
  ├─ auth_identities
  ├─ user_role_memberships
  ├─ uploader_profiles
  ├─ admin_role_assignments
  └─ observer_profiles

uploader_profiles
  └─ assets
      ├─ asset_tags
      ├─ asset_files
      ├─ asset_review_events
      └─ certification_records

orders
  ├─ order_items
  │   ├─ authorization_records
  │   └─ revenue_records
  └─ payments

payments
  └─ refunds

authorization_records
  ├─ download_links
  └─ downloads

platform_metric_snapshots
  └─ observer_share_records
```
