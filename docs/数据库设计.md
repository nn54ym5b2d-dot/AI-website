版本：v3.2
日期：2026-07-19

## 1. 设计结论

第一版数据库使用 TencentDB for PostgreSQL。T007 只细化数据模型，不连接真实数据库、不创建生产数据表、不写完整业务接口。

Robert 已确认 T009 起本地开发和测试使用 **Docker Compose + PostgreSQL 16 + Prisma**。T009 负责把本文件的数据模型落实为 `schema.prisma`、首个 migration、可重复初始化的本地数据库和非真实最小测试数据；生产环境仍使用 TencentDB for PostgreSQL，并在 T017 接入。

T008 补充说明：核心 API 还需要会话、手机号/邮箱验证码挑战、幂等记录、短期上传意图和第三方回调去重等运行时支撑数据。这些不是新的业务主体，不改变本文件已确认的用户、素材、交易、授权、收益和观察员关系；T009、T011、T013 实现时应按 `docs/API.md` 当前版本增加最小必要 Prisma 模型、受控缓存或等价持久化方案，并通过对应 migration 和 PR 审核。

T009 已实现 `users`、`auth_identities`、`legal_document_versions`、`user_terms_acceptances`、`user_role_memberships`、`uploader_profiles`、`invite_codes`、`admin_role_assignments`、`observer_profiles`、`user_sessions` 和 `auth_challenges`。T010 已按本文件第 5 节实现 `assets`、`asset_tags` 和 `asset_files`，并用独立 migration、可重复 seed 和真实 PostgreSQL 集成测试验证。交易、授权、收益和观察员汇总表继续按 T013-T015 的业务实现顺序加入后续 migration。

实名认证延期到 MVP 后，不在本版数据库中建立实名资料表。人物素材仍必须保存必要证明材料；物件/道具和场景素材不要求人物证明材料。

推荐后续实现方式：**Prisma + PostgreSQL**。

原因：

- Prisma schema 对新项目更直观，适合 Robert 后续阅读和 Codex 维护。
- Prisma Migrate 能管理迁移记录，方便回滚和审查。
- 当前业务以结构化表关系为主，Prisma 足够覆盖用户、素材、订单、授权、收益和权限。
- 对复杂统计、部分索引、物化视图或高阶 SQL，可在后续用原生 SQL migration 补充。

暂不建议第一版直接使用 Drizzle 作为主方案。Drizzle 更贴近 SQL，但对当前新手管理和快速推进不如 Prisma 直观。

## 2. 全局建模原则

- 主键统一使用 `uuid`。
- 金额统一用整数分保存，例如 `amount_cents`，币种字段默认 `CNY`。
- 所有核心表保留 `created_at`、`updated_at`；需要软删除的表增加 `deleted_at`。
- 文件表只保存腾讯云 COS 的 `bucket`、`region`、`object_key`、哈希和元数据，不保存公开原始文件地址。
- 原始素材、人物证明材料、认证证书文件和购买 ZIP 始终私有；公开预览只允许使用异步生成的独立水印衍生图或缩略图。
- 水印预览图/缩略图与原文件必须使用不同 `object_key`，并在元数据中记录来源文件、处理状态和水印模板版本；CDN 地址不得反推出原文件对象。
- 外部观察员只能看汇总表或脱敏字段，不能直接查用户、支付流水、原文件、下载链接、内部日志。
- 后续接口层必须按角色检查权限，数据库字段设计不能替代权限控制。

## 4. 用户与身份

### users

平台用户基础表。一个用户可以同时是购买用户、上传者、管理员或外部观察员，具体角色放在角色表中。

新购买用户以已验证手机号作为主身份；邮箱和微信是绑定在同一 `users` 记录上的辅助登录身份。历史后台或观察员测试账号可以保留邮箱主身份，不要求通过本轮规则伪造手机号。

| 字段 | 类型建议 | 说明 |
|---|---|---|
| `id` | uuid pk | 用户 ID |
| `display_name` | text | 显示名称；手机号/邮箱首次建号时由服务端随机生成，微信首次建号可使用经清理的授权昵称 |
| `email` | text nullable unique | 邮箱，敏感字段 |
| `phone` | text nullable unique | 手机号，敏感字段 |
| `avatar_url` | text nullable | 头像地址，可选 |
| `primary_login_method` | auth_provider | 主要登录方式 |
| `status` | user_status | 用户状态 |
| `last_login_at` | timestamptz nullable | 最近登录时间 |
| `created_at` | timestamptz | 创建时间 |
| `updated_at` | timestamptz | 更新时间 |
| `deleted_at` | timestamptz nullable | 软删除时间 |

索引：

- `unique(email)`，忽略 null。
- `unique(phone)`，忽略 null。
- `index(status, created_at)`。

外部观察员不可见：`email`、`phone`、登录时间、删除状态。

`display_name` 只用于展示，不是登录凭证，也不要求唯一。登录/绑定已有账号时不得修改该字段；后续允许用户改昵称时应使用独立资料更新接口。

### auth_identities

手机号、邮箱、微信等登录身份表。

邮箱注册必须同时验证手机号；首次微信登录也必须验证手机号。绑定前按 `(provider, provider_subject)` 查询现有归属：两种身份均未使用时创建新用户，只有一种已存在时绑定到该用户，两种身份分别属于不同用户时禁止自动合并。

| 字段 | 类型建议 | 说明 |
|---|---|---|
| `id` | uuid pk | 登录身份 ID |
| `user_id` | uuid fk users.id | 关联用户 |
| `provider` | auth_provider | 登录方式 |
| `provider_subject` | text | 登录方式下的唯一标识，如手机号、邮箱、微信 openid/unionid |
| `is_verified` | boolean | 是否已验证 |
| `verified_at` | timestamptz nullable | 验证时间 |
| `metadata` | jsonb | 第三方返回的非敏感补充信息 |
| `created_at` | timestamptz | 创建时间 |
| `updated_at` | timestamptz | 更新时间 |

索引：

- `unique(provider, provider_subject)`。
- `index(user_id, provider)`。

外部观察员不可见整张表。

## 5. 素材、文件、审核与认证

### assets

素材主表。

| 字段 | 类型建议 | 说明 |
|---|---|---|
| `id` | uuid pk | 素材 ID |
| `uploader_profile_id` | uuid fk uploader_profiles.id | 上传者 |

| `currency` | text | 默认 CNY |
| `rejection_reason` | text nullable | 驳回原因 |
| `submitted_at` | timestamptz nullable | 提交审核时间 |
| `reviewed_by_user_id` | uuid fk users.id nullable | 审核管理员 |
| `reviewed_at` | timestamptz nullable | 审核时间 |

业务约束：

- `listing_status = listed` 时，必须满足 `review_status = approved` 且 `certification_status = certified`。
- `asset_type = person` 时，必须存在至少一个 `asset_files.file_type = person_proof`。
- 所有素材最终上架前必须存在认证证书文件。
- 首版创建素材时，人物和场景默认 `price_cents = 5000`，物件/道具默认 `price_cents = 1000`；后续由系统设置统一调整，不允许上传者自行定价。

索引：

- `index(asset_type, listing_status, certification_status)`。
- `index(uploader_profile_id, created_at)`。
- `index(review_status, created_at)`。

### asset_files

素材文件表。所有原文件、预览图、缩略图、人物证明材料、认证证书文件和由原文件清单生成的 ZIP 下载包都放这里。

| 字段 | 类型建议 | 说明 |
|---|---|---|
| `id` | uuid pk | 文件 ID |
| `asset_id` | uuid fk assets.id | 关联素材 |
| `uploaded_by_user_id` | uuid fk users.id | 上传人 |
| `file_type` | asset_file_type | 文件类型 |
| `access_scope` | file_access_scope | 访问范围 |
| `cos_bucket` | text | COS Bucket |
| `cos_region` | text | COS 地域 |
| `cos_object_key` | text | COS object key，不是公开 URL |
| `file_hash` | text | 文件哈希 |
| `file_size_bytes` | bigint | 文件大小 |
| `mime_type` | text | MIME 类型 |
| `width` | integer nullable | 图片宽度 |
| `height` | integer nullable | 图片高度 |
| `metadata` | jsonb | EXIF、处理状态、来源文件 ID、水印模板版本、处理 provider/任务摘要等补充信息；不保存签名 URL |
| `created_at` | timestamptz | 创建时间 |
| `deleted_at` | timestamptz nullable | 软删除时间 |

索引：

- `index(asset_id, file_type)`。
- `unique(cos_bucket, cos_region, cos_object_key)`。
- `index(file_hash)`。

安全边界：

- 原文件、人物证明材料、认证证书文件：`access_scope = private` 或 `signed_download_only`。
- ZIP 下载包：`file_type = download_bundle`、`access_scope = signed_download_only`；`metadata` 保存包版本和原文件 ID/哈希清单，不保存签名 URL。
- 水印预览图、缩略图：必须作为独立 `asset_files` 记录，由私有原文件通过受控处理任务生成；可设为 `public_preview` 并经 CDN 分发，但不得暴露或替代路径访问原文件。
- 素材上架前至少一张水印预览图的处理状态必须为 `ready`；处理失败时不得回退为公开原文件。
- CDN URL 和临时签名 URL 不持久化；数据库只保存 COS 定位信息与衍生关系元数据。
- 外部观察员不可见 `cos_object_key`、`file_hash`、证明材料和证书文件路径。

### download_links

限时下载入口记录表。用户侧资格默认有效 365 天并依赖登录会话校验，不使用可转发的原始 token，也不保存完整 COS 临时 URL。多角度原文件以一个 ZIP 包交付。
