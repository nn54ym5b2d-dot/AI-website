当前状态：后续已按 Robert 明确指令接入 GitHub 仓库、GitHub Actions 和 Notion 总控页；腾讯云、COS、支付、认证接口、Vercel、Apifox 仍未接入。
来源：Robert 指令和当时本地状态

### D009：第一版技术栈方向

日期：2026-06-16
决策：第一版技术方向采用 Next.js + PostgreSQL + 腾讯云 COS 对象存储。
原因：Robert 已确认“技术栈建议没问题，用这个”；该方案适合 Web 网站、管理后台、订单、授权、收益和权限系统。
影响：后续代码框架优先按 Next.js 全栈项目设计。

### D010：中国大陆访问与备案

日期：2026-06-16
决策：网站面向中国大陆访问，需要准备备案；生产部署使用腾讯云大陆节点，不默认使用 Vercel。
原因：Vercel 官方区域列表包含香港、新加坡、东京等区域，但未列出中国大陆计算区域；阿里云和腾讯云均提供备案流程说明。
影响：生产部署方案固定为腾讯云；Vercel 仅保留为早期预览或海外演示备选。
参考：<https://vercel.com/docs/regions>、<https://help.aliyun.com/zh/icp-filing/>、<https://cloud.tencent.com/document/product/243>

### D013：认证证书保存

日期：2026-06-16
决策：平台需要保存认证证书文件，并保存证书编号、凭证、认证状态、认证时间等相关信息。
来源：Robert 2026-06-16 指令

### D014：外部观察员字段和导出限制

日期：2026-06-16
决策：外部观察员是网站合作方，可以只读查看平台上传量、下载量、收益情况和合作方分成相关字段；不允许导出任何后台数据。
原因：外部观察员收益分成基于平台上传量、下载量和收益情况计算。
影响：后台需要提供外部观察员专用汇总视图，同时隐藏个人隐私、支付敏感信息、原始文件地址和内部日志。
