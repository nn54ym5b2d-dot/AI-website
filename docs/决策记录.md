### D002：第一版素材范围

日期：2026-06-16  
决策：第一版以图片/多角度照片素材为主，覆盖人物、物件/道具、场景。视频、3D、素材包后续扩展。  
来源：`虚拟素材网站产品文档_v1.docx`

### D003：上传者准入方式

日期：2026-06-16  
决策：第一版上传者通过邀请码注册或激活，不开放无邀请码上传者注册。  
来源：`虚拟素材网站产品文档_v1.docx`

### D004：购买方式

日期：2026-06-16  
决策：第一版采用单次购买模式，一个订单可以购买多个素材。  
来源：`虚拟素材网站产品文档_v1.docx`

### D005：认证上传费

日期：2026-06-16  
决策：认证上传费默认每份素材 10 元，后台预留字段，后续可调整。初审未通过时原路退款。  
来源：`虚拟素材网站产品文档_v1.docx`

### D006：授权记录

日期：2026-06-16  
决策：支付成功后不仅开放下载权限，还必须生成授权记录。  
来源：`虚拟素材网站产品文档_v1.docx`

### D007：外部观察员权限

日期：2026-06-16  
决策：外部观察员只读查看平台上传量、下载量、收益汇总和合作方分成相关数据；不能编辑、审核、退款、录入认证、修改价格、管理账号或导出任何后台数据。  
来源：`虚拟素材网站产品文档_v1.docx`

### D008：本地工作系统优先

日期：2026-06-16  
决策：当前先建立本地 docs 和 Codex 工作规则，不擅自连接 GitHub、Notion、Vercel 等外部系统。  
来源：Robert 指令和当前本地状态

### D009：第一版技术栈方向

日期：2026-06-16  
决策：第一版技术方向采用 Next.js + PostgreSQL + 腾讯云 COS 对象存储。  
原因：Robert 已确认“技术栈建议没问题，用这个”；该方案适合 Web 网站、管理后台、订单、授权、收益和权限系统。  
影响：后续代码框架优先按 Next.js 全栈项目设计。

### D010：中国大陆访问与备案

日期：2026-06-16  
决策：网站面向中国大陆访问，需要准备备案；生产部署使用腾讯云大陆节点，不默认使用 Vercel。  
原因：Vercel 官方区域列表包含香港、新加坡、东京等区域，但未列出中国大陆计算区域；阿里云和腾讯云均提供备案流程说明。  
影响：生产部署方案固定为腾讯云；Vercel 仅保留为早期预览或海外演示备选。  
参考：<https://vercel.com/docs/regions>、<https://help.aliyun.com/zh/icp-filing/>、<https://cloud.tencent.com/document/product/243>

### D011：第一版支付方式

日期：2026-06-16  
决策：第一版支持微信支付和支付宝。  
来源：Robert 2026-06-16 指令

### D012：第一版登录方式

日期：2026-06-16  
决策：第一版采用手机号、邮箱、微信组合登录方式。  
来源：Robert 2026-06-16 指令

### D013：认证证书保存

日期：2026-06-16  
决策：平台需要保存认证证书文件，并保存证书编号、凭证、认证状态、认证时间等相关信息。  
来源：Robert 2026-06-16 指令

### D014：外部观察员字段和导出限制

日期：2026-06-16  
决策：外部观察员是网站合作方，可以只读查看平台上传量、下载量、收益情况和合作方分成相关字段；不允许导出任何后台数据。  
原因：外部观察员收益分成基于平台上传量、下载量和收益情况计算。  
影响：后台需要提供外部观察员专用汇总视图，同时隐藏个人隐私、支付敏感信息、原始文件地址和内部日志。

### D015：对象存储服务商

日期：2026-06-16  
决策：第一版对象存储使用腾讯云 COS。  
来源：Robert 2026-06-16 指令  
影响：素材原文件、预览图、缩略图、人物必要证明材料和认证证书文件后续均按腾讯云 COS 设计存储路径、权限和临时访问链接。

### D016：生产部署服务商

日期：2026-06-16  
决策：第一版生产部署使用腾讯云。  
来源：Robert 2026-06-16 指令  
影响：后续部署方案优先按腾讯云 CVM/轻量应用服务器、TencentDB for PostgreSQL、COS、CDN、HTTPS 和腾讯云备案流程设计。
