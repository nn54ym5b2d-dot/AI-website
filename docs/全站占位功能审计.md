# 全站占位功能审计

版本：v1.1
日期：2026-07-20
状态：审计完成；T012A 已解除前置阻塞并转为 Ready，T013-T017 责任已写入规划

## 1. 审计目标与判定标准

本次审计覆盖当前 `app/` 路由、页面可见文案、按钮与链接、对应 API/Prisma 数据来源，以及 T012A-T017 的任务范围。目的不是临时扩展已经完成的 T012，而是确保第一版已经露出的入口最终都有真实业务闭环，或明确保留为外部服务边界。

以下任一情况视为“占位未落地”：

- 页面使用硬编码演示记录、演示数量或演示金额，却看起来像真实业务数据。
- 按钮可点击但只改变本地组件状态，不保存数据、不调用 API，也没有明确“尚未开放”的不可操作状态。
- 页面只有流程说明或字段清单，没有真实查询、提交、权限和状态流转。
- 已有路由和导航入口，但对应的后续任务没有逐项写明数据、API、页面和验收责任。
- 页面文案宣称“精选”“认证可查”等能力，但实际数据只能证明更弱的含义。

明确标记为 `local_test`、非真实 seed 或“真实外部服务尚未接入”的实现不自动视为遗漏；只要本地业务闭环真实可验证，并已由 T017 明确负责替换生产 provider，即属于受控开发边界。

## 2. 审计结论

当前 T009-T012 已完成的身份、素材浏览、上传、审核与认证记录主链路是真实 PostgreSQL/Prisma 业务实现，不是页面占位。但全站仍有三类缺口：

1. **原规划没有明确接手的现有入口**：邀请码管理、系统设置、上传者个人页、公开认证摘要、法律文档展示和首页“本周精选”语义。
2. **后续任务方向正确但范围写得过粗**：结算页、购买/下载/授权/收益个人页，以及订单、支付、授权、收益、用户和观察员后台。
3. **最终质量门槛缺失**：没有明确要求在 MVP 收口时删除通用演示页、无行为按钮和废弃原型组件。

因此新增 T012A 作为 T012 与 T013 之间的基础管理和现有入口补全任务；T013 从 Ready 调整为 Blocked，待 T012A 完成后开始。T014-T017 补齐逐路由责任，T016 增加“全站无伪功能占位”总验收门槛。

## 4. T012A 范围

任务名称：**T012A 后台基础管理与现有入口补全**。

前置：T012 验收、PR 合并并转 Done（已满足）。后置：T013。

T012A 必须完成：

- 邀请码管理真实闭环。
- 系统设置真实闭环，并为 T013/T014 提供服务端价格、认证费、分成比例和下载资格配置来源。
- 上传者“我的上传 / 审核状态 / 上传者资料”和个人中心已落地数据的真实化。
- 管理员用户列表/详情与角色摘要的只读基础；敏感角色修改留给 T015。
- 素材公开认证安全摘要。
- 首页“本周精选”语义修正。
- 版本化条款、隐私和商业授权展示入口；当前非正式文本必须明确标记。
- 对仍等待 T013-T015 的正式路由显示准确的“尚未开放/对应任务”状态，移除会误导用户的假记录与无行为按钮。

T012A 不做：

- 不开发订单支付、下载 ZIP、收益结算或观察员汇总。
- 不接入真实短信、邮件、微信、COS、CDN、支付商户或政府认证网站。
- 不在 T012A 开放管理员角色变更、观察员账号生命周期等 T015 高权限写操作。

## 5. 系统设置生效规则

- 设置值由服务端读取，前端提交的价格、费率和资格天数不能作为权威值。
- 新订单、新认证费和新授权按创建时的有效设置生成不可变快照；后续修改设置不得重写历史订单、授权、收益或退款金额。
- 已上架但尚未进入订单的素材价格使用当前有效设置；订单创建后固定 `order_items` 价格快照。
- 上传者 80%、平台 20%、观察员 0% 是首版默认 seed，不是散落在页面中的永久硬编码。
- 设置修改必须校验范围、记录操作者和前后值；只有超级管理员可写。

## 6. MVP 无占位完成门槛

T016 完成前必须进行一次从导航入口出发的逐路由复核：

- 每个第一版 P0/P1 入口要么有真实本地数据库/API/权限闭环，要么明确属于 T017 的生产 provider 替换边界。
- 正式导航可达页面不得包含伪造的订单、金额、收益、用户或素材记录。
- 可点击控件必须执行实际行为；尚未开放的行为使用不可点击状态和明确说明。
- 未知动态 section 返回 404，不得落入通用假数据页面。
- 桌面与 390px 页面无横向溢出、主要操作可完成、控制台无错误；交易与权限关键路径还需自动化测试。
