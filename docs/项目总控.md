## 2. 项目一句话说明

源素库是一个面向中国大陆 AI 视频、游戏、短剧广告和虚拟内容制作方的数字素材交易平台。

第一版做 Web 网站 + 管理后台，支持人物、物件/道具、场景图片素材的认证上传、审核上架、付费购买、下载授权、收益记录和后台权限管理。

## 3. 第一版产品范围

第一版做：

- Web 网站。
- 管理后台。
- 电脑和手机浏览器访问。
- 人物、物件/道具、场景图片素材。
- 上传者邀请码注册或激活。
- 素材认证上传、初审、认证证书保存和上架。
- 单次购买，一个订单可购买多个素材。
- 支付成功后生成授权记录和限时下载入口。
- 上传者收益记录。
- 外部观察员只读查看平台上传量、下载量、收益汇总和合作方分成相关数据。

第一版不做：

- App。
- 微信小程序。
- 会员订阅。
- 自动提现和复杂分账。
- 复杂社区功能。
- 企业授权系统。
- 真实版权认证接口自动对接。

## 10. 同步规则

以后每完成一个任务，至少同步以下位置：

1. Notion 总控页：给 Robert 看当前阶段、进度和下一步。
2. `docs/项目总控.md`：作为 Notion 总控页的本地同步源。
3. `docs/任务清单.md`：更新 T 编号任务状态。
4. GitHub Issue：更新具体任务卡状态和结果。
5. 如规则、决策、页面、数据库或 API 发生变化，同步更新对应 `docs/` 文件。

当前采用“Codex 半自动同步”：

- Robert 下达任务或确认验收。
- Codex 修改代码和文档。
- Codex 提交/推送到 GitHub。
- Codex 更新本地总控、任务清单和相关 docs。
- Codex 更新或创建 Notion 页面。
- GitHub Issues / Projects 接入后，再同步任务状态。

暂不做完全自动同步，因为 Notion API、GitHub Projects 权限和自动化规则还未全部确认。等项目流程稳定后，再考虑 GitHub Issue 关闭后自动同步 Notion。
