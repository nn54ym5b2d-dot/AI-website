版本：v0.1

## 1. 文档用途

本文件是 T009A `源素库 MVP 视觉设计基线与核心页面原型` 的本地事实来源，用于记录设计简报、视觉方向选择、设计 token、公共组件、响应式规则、代表页面和验收结论。

当前版本只确认任务范围和执行顺序，不代表三套视觉方向、最终颜色、字体、Logo 或正式页面视觉已经完成。

T009A 完成后，T010-T016 必须优先复用本文件确认的颜色、字体、间距、图片比例和公共组件，不在每个任务中重新定义基础风格。

## 2. 已固定的产品与结构边界

- 产品：面向中国大陆 AI 视频、游戏、短剧广告和虚拟内容制作方的数字素材交易平台。
- 核心素材：人物、物件/道具、场景图片和多角度照片。
- 结构基础：继续使用 T006 已确认的 `/`、`/materials`、`/search`、`/materials/[assetId]`、`/login`、`/register`、`/account/*`、`/upload`、`/admin/*` 和 `/observer`。
- 大众入口：公开网站突出访客、购买用户和上传者；管理后台和外部观察员继续作为低显眼内部入口。
- 身份基础：保留 T009 已实现的会话、CSRF、条款接受、邀请码激活和角色守卫，不因视觉原型削弱权限边界。
- 数据边界：原型使用明确标注的非真实演示素材和数据，不伪装为真实数据库、支付、COS、认证或生产服务。

## 3. 默认设计简报

Robert 尚未提供参考网站或截图时，使用以下默认简报启动三套视觉方向：

- 目标感受：专业、可信、克制、具有创意、突出素材图片。
- 主要用户：个人 AI 视频创作者、游戏和虚拟内容制作方、短剧广告团队、设计工作室。
- 核心目标：让用户快速理解素材质量、类型、认证状态、价格和商业授权可信度。
- 内容优先级：素材图片 > 素材名称/类型 > 认证状态 > 价格和购买入口 > 辅助说明。
- 默认避免：廉价电商感、过度赛博、后台感过强、满屏小卡片、装饰压过素材、缺少留白和层级。

Robert 后续可提供 2-5 个参考网站、截图、品牌、颜色或明确不喜欢的风格；参考资料只用于 T009A，不自动改变产品规则或页面结构。

## 4. 执行顺序

1. 整理参考资料或采用默认设计简报。
2. 使用相同页面内容生成三套差异明确的视觉方向；此时不修改正式页面代码。
3. Robert 选择一套主方向，可对选中方向提出有限调整。
4. 将选中方向转为颜色、字体、字号、间距、圆角、边框、阴影、图片比例、图标和状态规则。
5. 使用真实感但非真实业务的演示素材，完成六组代表页面的响应式可运行原型。
6. 检查核心交互、桌面端、手机端、可访问性和与选中视觉稿的可见差异。
7. 通过分支、PR 和 GitHub Actions 交给 Robert 验收。
8. Robert 验收并合并后，T009A 进入 Done，T010 从 Blocked 调整为 Ready。

## 5. 三套视觉方向

三套方向必须使用相同的首页、素材卡片和详情信息内容，避免因为内容不同而无法比较。

| 方向 | 探索重点 | 当前状态 |
|---|---|---|
| A：专业创作市场 | 浅色、克制、高可信度，素材图片和购买信息优先 | 待生成 |
| B：AI 影像实验室 | 深色或冷色、沉浸感更强，突出 AI 创作氛围 | 待生成 |
| C：数字素材档案馆 | 温暖中性色、编辑排版和收藏感，强调认证与精选 | 待生成 |

最终必须选择一套主方向。可以从其他方向借用少量明确元素，但不把三套平均混合，也不同时开发三套正式页面。

## 6. 代表页面原型范围

| 页面组 | 代表内容 | 必须可操作的核心状态 |
|---|---|---|
| 首页 | 平台定位、搜索入口、三类素材、精选素材、上传者入口 | 主导航、搜索入口、素材卡片入口 |
| 素材浏览/搜索 | 素材网格、关键词、类型、价格、认证和排序 | 搜索、筛选、排序、卡片跳转 |
| 素材详情 | 多角度预览、价格、认证、授权说明和购买入口 | 图片切换、信息层级、主要购买按钮 |
| 登录/注册 | 手机号/邮箱、验证码、条款接受和邀请码提示 | 模式切换、输入、错误/禁用/加载状态 |
| 个人中心/上传 | 购买、下载、授权、上传状态、收益和上传入口 | 导航、标签/状态切换、主要操作 |
| 管理后台 | 侧边导航、数据概览、待审核列表和权限提示 | 导航、表格/列表、筛选和状态标记 |

`/observer` 在 T009A 中复用后台的视觉语言，但必须保持只读、汇总和不允许导出的既有边界；不要求单独完成全部观察员数据页面。

## 7. 响应式与交互要求

- 至少验证约 1440px 桌面宽度和约 390px 手机宽度。
- 页面不能出现非预期横向滚动、关键按钮遮挡、图片拉伸或文字溢出。
- 素材图片必须保持统一比例和合理裁切，不使用空白占位框代替最终可见素材。
- 核心导航、链接、标签、筛选、主要按钮、输入框和表单状态必须可操作。
- 原型不接真实业务 API 时，所有演示数据必须集中管理并明确标识为非真实。
- 使用键盘时应能看到焦点；输入框有标签；错误不能只依赖颜色表达；手机端主要点击区域应足够清楚。

以下字段在 Robert 选定方向后填写；未选定前不得把当前蓝灰色占位样式记录为最终规范。

| 类别 | 需要记录 | 当前状态 |

| 颜色 | 品牌色、背景、表面、主文字、次文字、边框、成功、警告、错误 | 待选定 |
| 字体 | 中文/英文栈、标题与正文字重、数字价格样式 | 待选定 |
| 字号 | 页面标题、分区标题、卡片标题、正文、辅助文字 | 待选定 |
| 间距 | 页面边距、分区间距、卡片内边距、控件间距 | 待选定 |
| 形状 | 圆角、边框、阴影和分隔线 | 待选定 |
| 图片 | 素材卡片、详情主图、缩略图比例和裁切策略 | 待选定 |
| 组件 | 按钮、输入框、筛选器、素材卡片、认证标签、价格区、表格和侧边栏 | 待选定 |

选定后的 token 应集中维护在 `tailwind.config.ts`、`app/globals.css` 和公共组件中，避免页面散落重复颜色或尺寸。

## 9. 不做范围

- 不在选择视觉方向前修改正式页面样式。
- 不接素材数据库、真实素材查询 API、支付、COS、上传、审核、下载或认证服务。
- 不改变 T006 路由、角色入口和信息结构。
- 不改变 T009 身份和权限实现。
- 不完成所有后台子页面，不制作完整品牌手册，不要求本任务确定最终 Logo。
- 不新增重型 UI 工具或设计平台作为必需依赖；如后续需要 Figma 或组件库，另行确认。

## 10. 验收标准

- Robert 已从三套视觉方向中选择一套明确的主方向。
- 本文件已补齐颜色、字体、间距、图片比例、组件和状态规范。
- 六组代表页面使用统一视觉，桌面端和手机端均可正常浏览。
- 素材图片是视觉重点，不被装饰或后台式信息密度压过。
- 核心导航和基础交互可操作，演示素材与数据标识真实准确。
- 登录注册和既有角色权限没有被视觉改动破坏。
- 未提前接入 T010-T017 的真实业务功能或外部服务。
- T010 可以直接复用本任务确认的视觉规则和公共组件。

## 11. 检查要求

- 同视口对比选中视觉稿与页面实现，检查布局、裁切、留白、字体、颜色、边框和圆角。
- 检查桌面端与手机端六组代表页面。
- 检查键盘焦点、颜色对比度、表单标签、按钮可点击区域和常见状态。
- 运行 `npm run lint`。
- 运行 `npm run typecheck`。
- 运行 `npm test`。
- 运行 `npm run build`。
- GitHub Actions `verify` 通过。
