# docs 目录说明

本目录是 Codex 后续开发、整理、测试和维护项目时的主要依据。

## 1. 文件索引

| 文件 | 用途 |
|---|---|
| `项目总控.md` | Robert 和 Codex 共用的总控入口，集中查看目录、当前结论、协作连接状态、大任务路线图、待确认问题和同步规则 |
| `PRD.md` | 产品定位、目标用户、MVP 范围和核心流程 |
| `业务规则.md` | 上传、认证、审核、购买、授权、收益、退款等规则 |
| `页面清单.md` | 前台、个人中心、管理后台页面列表 |
| `全站占位功能审计.md` | 逐路由记录当前演示/无行为入口、真实数据缺口、T012A-T017 归属和 MVP 无占位完成门槛 |
| `角色权限.md` | 用户、上传者、管理员、财务、外部观察员权限边界 |
| `视觉设计基线.md` | T009A 设计简报、视觉方向流程、响应式原型范围、视觉规范字段和验收边界 |
| `技术方案.md` | 初版技术栈、系统模块和部署方式草案 |
| `数据库设计.md` | 核心数据表、字段、关系、敏感字段和外部观察员汇总模型 |
| `API.md` | T008 已细化的前后端接口合同 |
| `测试验收清单.md` | Robert 验收和 Codex 自查用例 |
| `任务清单.md` | 第一阶段到 MVP 的任务拆分 |
| `GitHub Issue 草稿.md` | T006-T017 与 T009A 的任务卡原稿和 GitHub Issue 映射 |
| `决策记录.md` | 已确认决策、待确认问题和变更记录 |
| `工具协作流程规划.md` | 工具、GitHub、Codex、腾讯云部署等协作流程 |

## 2. 使用规则

- 项目当前状态和推进顺序先看 `项目总控.md`。
- 产品规则以 `PRD.md` 和 `业务规则.md` 为主。
- 页面和后台开发以 `页面清单.md` 和 `角色权限.md` 为主。
- 视觉方向、设计 token、响应式原型和公共 UI 组件以 `视觉设计基线.md` v1.1 为主；T009A 已完成，T010 起必须优先复用。
- 技术实现以 `技术方案.md`、`数据库设计.md`、`API.md` 为主。
- 如果内容冲突，先查看 `决策记录.md`，再向 Robert 确认。
- 每次完成任务或新增重要规则，都要同步更新 `项目总控.md`、相关 docs 文件和 `决策记录.md`。
