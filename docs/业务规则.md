# 业务规则

版本：v2.1
日期：2026-07-22

## 1. 上传者规则

- 上传者必须通过邀请码注册或激活。
- 公开业务账号注册后默认是购买者；开通上传资格后必须同时持有 `buyer + uploader`，上传资格不替换购买者身份。
- 上传者继续拥有购买、订单、授权和下载能力，同一账号不需要注册第二套购买账号。
- `/account` 购买者中心和 `/account/uploader` 上传者中心分别展示各自业务入口，但共用同一账号与登录状态。
- 同一个上传者账号可以上传人物、物件/道具和场景素材。
- 第一版不开放无邀请码上传者注册。
- 邀请码只能激活一个上传者账号；一旦已使用或已关联上传者资料，不得通过后台操作、测试 seed 或状态修复重新开放。
- 企业用户第一版可先合并为购买用户；企业账户、企业授权、批量采购和对公支付放到后续版本。

## 2. 素材上传规则

- 第一版只处理图片/多角度照片素材。
- 每份素材需要保存全部角度原文件、由系统异步生成的独立水印预览图/缩略图、文件信息和哈希值；公开页面和 CDN 只访问水印衍生图，购买后把全部有效原文件打包为一个私有 ZIP 交付。
- 上传者提交素材时需要填写素材名称和类型；说明可选填。
- 只有人物素材需要提交必要证明材料。
- 物件/道具和场景素材可以选填来源、版权或所有权证明；选填材料与人物证明一样保持私有，但不是提交前置条件。
- 所有素材，包括人物/物件/场景，均需获得认证证书后才能上架。
- 实名认证延期到 MVP 后；第一版不建立实名认证流程或保存实名资料。

## 3. 认证上传费规则

- 认证上传费默认每份素材 10 元。
- 10 元为后台可配置字段，后续可调整。
- 上传提交时支付认证上传费。
- 初审未通过时，当场创建认证费原路退款待办；退款成功只能由已验证的支付平台回调或主动查询确认，不能由页面或管理员直接写成成功。
- 初审通过后进入政府官方网站认证流程。
- 认证上传费长期独立记录在 `certification_fee_charges`，不并入素材购买订单。

## 4. 审核和认证规则

| 阶段 | 规则 |
|---|---|
| 初审 | 平台检查素材基础质量、类型、信息完整度和明显违规情况 |
| 政府认证 | 初审通过后跳转政府官方网站完成政府背书认证 |
| 认证记录 | 平台保存证书文件，并记录证书编号、凭证、认证状态和认证时间 |
| 上架 | 审核通过且认证完成后才允许上架 |

T012 已把上述规则落实为本地数据库状态机：审核通过创建人工认证记录；认证通过必须填写证书编号并关联私有证书文件；上架还要求存在可用水印预览。审核、认证、上架/下架、基础信息编辑和敏感文件查看均记录操作日志。真实认证费退款执行属于 T013，真实 COS 和政府认证网站属于 T017。

当前待确认：

- 政府官方网站的具体名称、认证路径和证书样式。
- 认证异常后的重新提交规则。

## 5. 购买规则

- 第一版采用单次购买模式。
- 一个订单可以购买多个素材。
- 用户需登录后才能创建订单和支付。
- 支付成功后，订单内每个素材分别开放下载权限。
- 支付成功后，订单内每个素材分别生成授权记录。

## 6. 定价规则

- 平台统一定价，不由上传者自由定价。
- 第一版人物素材统一 50 元、场景素材统一 50 元、物件/道具素材统一 10 元。
- 后台系统设置保存素材分类价格，只有超级管理员可修改；运营和财务只读。
- 新订单的订单明细在创建时保存服务端当前价格快照，修改系统设置不得重写历史订单、授权、收益或退款金额。
- 新认证费记录同样保存创建时系统设置快照；T012A 已将上传草稿默认价格和认证费改为从服务端设置读取，客户端价格不作为权威值。
- 邀请码完整明文只在创建成功时展示一次，后续只显示掩码；只有未使用且未过期的邀请码可禁用，创建和禁用都必须写审计日志。
- 当前服务条款、隐私政策和商业授权为明确标记的本地草案/测试版本，不代表正式法律意见；T017 换入正式审阅文本并保留版本历史。
- 上传者不能自行定价。

## 7. 授权规则

用户付费购买后，默认可用于商业内容制作。

授权记录至少包括：

- 购买用户
- 素材编号
- 订单编号
- 统一商业授权说明
- 授权时间
- 素材认证状态
- 必要使用限制

禁止行为至少包括：

- 转售素材原文件
- 冒充素材权利人
- 违法用途
- 侵权用途
- 平台规则明确不允许的其他用途

- 统一商业授权协议的正式法律文本。
- 是否区分个人商用和企业商用。
- 第一版授权永久有效，不设置自然到期时间；退款、违规或争议处理等特殊情况可撤销，并记录原因。

## 8. 下载规则

- 支付成功后按当时系统设置生成默认有效 365 天的用户下载资格快照，后续改设置不影响已有授权。
- 一份多角度素材以一个 ZIP 包下载，ZIP 包含该素材的全部有效原文件。
- 原始文件地址不能直接公开。
- 下载记录需要保存用户、素材、订单、下载时间和下载状态。
- 每次通过有效下载入口取文件时，由服务端生成短时 COS 签名 URL，不保存或公开长期原文件地址。

平台下载资格（默认 365 天）与每次签发的 COS 临时 URL（建议 10 分钟）是两个不同期限，不得共用同一个设置或把 365 天长效地址直接暴露给用户。

## 9. 收益规则

- 第一版素材购买收益按上传者 80%、平台 20% 分配，后续可调整。
- 支付成功后，上传者收益记录增加。
- 第一版只记录收益，不做自动提现和复杂分账结算。
- 提现/结算状态放到后续版本扩展。
- 退款后需要冲正对应的上传者和平台收益记录。
- 素材购买退款可以只退订单中的部分明细，但每个被选中的订单明细必须整项退款，不允许拆分单个明细金额。

## 10. 状态规则

| 状态类型 | 建议状态 |
|---|---|
| 素材审核状态 | `draft`、`pending_review`、`approved`、`rejected` |
| 素材上架状态 | `unlisted`、`listed`、`delisted` |
| 认证状态 | `not_started`、`pending_payment`、`pending_review`、`certifying`、`certified`、`exception` |
| 订单状态 | `pending_payment`、`paid`、`cancelled`、`payment_failed`、`refunded`、`partial_refunded` |
| 授权状态 | `active`、`revoked` |
| 下载入口状态 | `active`、`expired`、`revoked` |
| 收益状态 | `recorded`、`reversed`、`settled_pending`、`settled` |
