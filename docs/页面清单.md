# 页面清单

版本：v2.7
日期：2026-07-19

## 1. T006 路由结构结论

T006 先建立结构优先的页面骨架，不做最终品牌视觉、不接入真实登录、支付、上传、数据库、COS 或认证服务。

公开网站主导航和首页显眼位置只展示面向大众或常规用户的入口：访客浏览、购买用户个人中心、上传者入口。管理后台和外部观察员不属于大众入口，只保留为页面底部低显眼内部入口。

第一版入口按角色分为：

| 角色/访问者 | 主要入口 | 说明 |
|---|---|---|
| 访客 | `/`、`/materials`、`/search`、`/materials/[assetId]` | 可浏览素材、搜索筛选、查看详情；购买前需要登录 |
| 购买用户 | `/account`、`/account/purchases`、`/account/downloads`、`/account/licenses`、`/checkout` | 查看购买、下载、授权记录；从素材详情进入订单支付 |
| 上传者 | `/upload`、`/account/uploads`、`/account/upload-status`、`/account/revenue`、`/account/uploader-profile` | 邀请码激活后上传素材，查看审核状态和收益 |
| 管理员 | `/admin` 及 `/admin/*` | 超级管理员、运营管理员、财务管理员共用后台入口；公开站点不显著展示 |
| 外部观察员 | `/observer` | 独立只读入口，查看平台上传量、下载量、收益汇总和合作方分成字段；首版分成比例为 0，不允许导出；公开站点不显著展示 |

T009A / Issue #21 已由 Robert 验收，PR #24 已合并，当前状态为 Done。方向 A（方向 1）的六组响应式原型和视觉基线 v1.1 已成为 T010-T016 的统一页面基础；T006 的路由和角色入口继续作为结构基础，T009 的身份和权限逻辑保持不变。

第一阶段已使用相同页面内容生成三套差异明确的视觉方向，且未在选择前修改正式页面代码。第二阶段按已选方向 A 完成首页、素材浏览/搜索、素材详情、登录/注册、个人中心/上传和管理后台六组代表页面的响应式可运行原型。至少检查约 1440px 桌面端和约 390px 手机端。

登录/注册页面的当前信息层级以 Robert 2026-07-17 最新确认为准：公开头部只保留一个“登录/注册”按钮，与“上传素材”共同进入 `/login`，但分别携带 `/` 和 `/upload` 回跳目标。顶部直接展示“手机号登录”与“微信登录”，手机号默认展示，不再显示“主要方式”；邮箱位于下方“其他方式”。手机号状态的条款提示和“登录/注册”主按钮位于分隔线之前，邮箱状态的对应控件仍跟随邮箱字段，避免按钮先于输入。手机号和邮箱验证码通过后由服务端判断登录或注册；新邮箱和首次微信必须绑定手机号。登录成功后普通入口回首页并在头部显示用户图标/昵称；上传入口进入 `/upload`，非上传者先激活邀请码，上传者直接进入提交页。`/register` 只作为兼容跳转。QQ 首版不展示，真实短信、邮件和微信二维码仍在 T017 接入。

设计简报和第一轮参考网站已确认：保留 A 专业创作市场、B AI 影像实验室、C 数字素材档案馆三套方向，三套共同保持正式、专业、简约、现实影像优先和认证可信。具体参考和借鉴边界见 `docs/视觉设计基线.md` 第 5 节。T010 已复用该基线完成首页素材入口、分类、搜索筛选和动态详情页真实数据闭环，并通过 PR #28 完成验收合并；真实支付、COS、图片处理、CDN 和原文件下载尚未接入。

## 9. T011 上传页实现状态

- `/upload` 已从视觉占位升级为真实 PostgreSQL 草稿工作区：人物素材使用独立表单，物件/道具与场景共用一套表单。
- 上传者可以创建草稿、选择最近草稿继续编辑、添加标签，并分别选择原文件和人物证明材料；非人物表单不显示或接受人物证明材料。
- 文件选择后由浏览器计算 SHA-256，再调用本地测试 provider 完成上传意图、幂等确认和衍生任务；页面明确提示当前只验证元数据、不保存真实文件、未接 COS。
- 页面展示原文件、人物证明、预览图和缩略图计数及处理状态；只有必需文件与水印衍生图全部就绪后才能提交。
- 提交后只进入“10 元认证费待支付”，在真实支付回调前不显示为已进入审核。桌面端和 390×844 手机端已完成布局与交互检查。
